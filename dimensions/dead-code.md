# 无效代码残料审查视角

## 检查清单

1. **未使用的变量** — 声明但从未被读取或引用的变量
2. **未使用的函数/方法** — 定义但从未被调用的函数或方法
3. **未使用的类/结构体** — 定义但从未被实例化或引用的类型
4. **死代码块** — 永远不会执行到的代码分支（如 return 后的代码、永假条件分支）
5. **残留注释** — 被注释掉的旧代码块（不是文档注释）
6. **未使用的依赖** — import/require 了但未使用的模块或包
7. **未使用的配置/常量** — 定义了但从未引用的配置项或常量

## 严重度判定

- **high**: 大段未使用的类/模块、影响构建体积或引入不必要依赖的死代码
- **medium**: 未使用的函数/方法、未使用的 import
- **low**: 单个未使用变量、残留注释

## 维度规则

- 谨慎判断以下场景，不要误报：框架约定的生命周期方法（如 React componentDidMount、Go init）、通过反射或动态调用的函数、对外暴露的 public API、条件编译代码
- 必须用 Grep 在整个项目中搜索引用，全局确认"未使用"后才能报告
- 每个问题必须给出文件路径 + 行号 + 代码片段作为证据
- 修复风险判定：删除未使用的变量/import/注释 = safe；删除未使用的函数/类 = local（需确认无反射调用）；删除整个模块 = structural
